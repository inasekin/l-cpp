# Задание 4

## Порядок вызовов

При `TrackedString x` цепочка: `TrackedString()` → `TrackedString("")` → `TrackedString(std::string(""))`.

Тела выполняются в обратном порядкеб, сначала самый глубокий конструктор, потом те что делегировали:

```
[строковый конструктор] id=1 — самый глубокий, выполнился первым
[char* конструктор делегированъ — вернулся в тело TrackedString(const char*)
[конструктор по умолчанию делегирован] — вернулся в тело TrackedString()
```

## Отличие от вспомогательного метода

Вспомогательный метод вызывается в теле конструктора после того как все поля уже инициализировались. 
Делегирующий конструктор полностью заменяет фазу инициализации, наши поля инициализируются целевым конструктором тело делегирующего выполняется позже

## Почему делегирование только в списке инициализации?

Если написать TrackedString() в теле это создаст новый rvalue, а текущий `this` не будет переинициализирован. 
Список инициализации это единственное место где можно управлять тем, как инициаизируется сам объект
