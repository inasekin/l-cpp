# Задание 6

## Количество вызовов конструкторов

- `push_back(TrackedString("temp"))` — 2 конструктора char* + перемещающий
- `push_back(ts)` — 1 конструктор копирующий плюс перемещения при reallocate
- `emplace_back("temp")` — 1 конструктор char*, объект строится прямо в векторе
- `emplace_back(TrackedString("temp"))` — 2 конструктора char* + перемещающий

## Когда создаются временные объекты?

При `push_back(TrackedString("temp"))` создаётся временный объект до того, как вызывается push_back. 

При `emplace_back("temp")` — никакого временного объекта нет, аргумент передаётся прямо конструктору

Также при каждом росте вектора существующие элементы перемещаются через перемещающий конструктор

## Почему emplace_back эффективнее?

`emplace_back` строит объект прямо в памяти вектора без промежуточного временного объекта и без move/copy это даёт выигрыш именно когда передаёшь аргументы конструтора, а не уже готовый объект. Если передать готовый объект, то разницы с push_back нет.
