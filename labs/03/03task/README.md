## Разница количества созданных объектов

В варианте А создаётся 2 объекта, в варианте B 1. 

В варианте А поле str_ сначала строится конструктором по умолчанию, потом создаётся временный объект и присваивается полю

## Почему без списка инициализации создаётся лишний объект?

Поля класса инициализируются до выполнения тела конструктора
Потом в теле конструктора ему присваивается значение

## Роль временного объекта

TrackedString(s) в теле конструктора это rvalue. Он создаётся чтобы передать значение в оператор присваивания и сразу уничтожается

## Порядок объявления полей и порядок в списке инициализации

Поля всегда иницилизируются в порядке их объявления в классе независимо от порядка записи в списке инициализации 

```
class Foo {
    int b;
    int a;
public:
    Foo(int x) : a(x), b(a) {} b инициализируется раньше a
};
```

b инициализируется первым, но пытается использовать a, который ещё не инициализирован

## Инициализация по умолчанию в списке инициализации

Примитивы остаются неинициализированными. 

Поля-классы инициализируются через конструктор по умолчанию
