# Задание 4

## Почему при передаче int* arr в функцию нельзя изменить сам указатель data в вызывающем коде?

Потому что указатель передается по значению, создается локальная копия адреса

## Что означает синтаксис int*& arr? Почему он позволяет изменить указатель?

Это ссылка на указатель. В функцию передается не копия адреса, а alias самой переменной из вызывающего кода. 
Любое действие с arr внутри функции это прямое взаимодействие с оригинальным указателем data

## Можно ли передать динамический массив «по значению»? Почему?

Нет, сам массив передать по значению в C++ нельзя. Можно передать только указатель на его первый элемент.
Если нам нужно скопировать массив по значению, то приется вручную создать новый массив и скопировать в него каждый элемент

## Что произойдёт, если в reallocateArray забыть обновить size?

Программа потеряет информацию о реальном размере нового массива, в результате выведется старое кол-во элементов

## Почему важно освобождать старый массив перед выделением нового в reallocateArray?

Если не вызвать delete[] arr перед присвоением нового адреса, область памяти, где лежал старый массив, останется занятой и это приведет к утечке памяти

## Какие риски возникают при передаче указателя без размера? Как этого избежать?

Выход за пределы памяти, чтение мусора или segmentation fault, тк функция не знает где, заканчивается массив
