# Задание 2

## Чем отличается const int* p от int* const p? Приведите примеры.

const int* p означает указатель на константное число (нельзя изменить значение через этот указатель), а int* const p означает константный указатель на целое число (нельзя изменить адрес)

Примеры из нашего кода с попытками изменения:
*p1 = 50; error: read-only variable is not assignable
p2 = &x; error: cannot assign to variable 'p2' with const-qualified type 'int *const'

## Почему нельзя изменить значение через const int*, даже если исходная переменная не const?

Нельзя изменить значение через const int*, потому что const в этом случае гарантирует, что компилятор будет рассматривать данные, на которые указывает указатель, как доступные только для чтения через этот указатель

## Можно ли передать неконстантную переменную в функцию, принимающую const int&? Почему?

Да, можно. Это называется неявным приведением типов. Компилятор разрешает это, тк передача в const int& гарантирует, что функция не будет изменять исходную переменную

## Что произойдёт, если попытаться присвоить новый адрес константному указателю (int* const)?

Произойдёт ошибка компиляции int* const p означает, что сам указатель p является константой и должен быть инициализирован при объявлении.

## Почему использование const в параметрах функций считается хорошей практикой?

Использование const в параметрах функций это хорошая практика тк повышает безопасность кода и читаемость + эффективность (оптимизация)

## Какой из способов передачи предпочтительнее для больших объектов, которые не нужно изменять: const T* или const T&? Почему?

Предпочтительнее использовать const T& (константную ссылку)

- const T& (константная ссылка) обычно предпочтительнее, тк синтаксис ее использования внутри функции чище (не требуется разименовние), она гарантированно не может быть nullptr, что предотвращает ошибки сегментации, и она также избегает копирования
- const T* (указатель на константу) используется только тогда, когда объект может быть необязательным
