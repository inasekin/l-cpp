# Задание 3

## Почему имя массива можно использовать как указатель? Что именно имя массива представляет?

Имя массива можно использовать как указатель, потому что оно представляет собой константный указатель на адрес первого элемента массива.
Имя представляет собой константный адрес начала области памяти, выделенной под массив

## Чем принципиально отличается статический массив от динамического с точки зрения времени жизни и размещения в памяти?

Статический: Размещается в стеке. Время жизни ограничено областью видимости (например, до конца функции)
Динамический: Размещается в куче. Живёт до тех пор, пока мы не вызовем delete[]

## Отличается ли размер статического и динамического массива? Почему?

Да, отличается sizeof для статического массива возвращает суммарный размер всех его элементов в байтах. Для динамического массива sizeof возвращает размер самого указателя, тк компилятор не знает размер выделенного блока в куче

## Почему компилятор знает размер статического массива, но не знает размер динамического?

Размер статического массива является частью его типа и известен на этапе компиляции
Память под динамический массив выделяется во время выполнения программы, и указатель просто хранит адрес не помня о количестве элементов

## Почему выражение &arr[1]-&arr[0] даёт результат 1, а не 4, ведь разница между адресами 4?

Вычитание указателей возвращает расстояние в количестве элементов того типа, на который они указывают, тк между соседними int ровно один int, результат равен 1

## Можно ли изменить значение staticArr (например, присвоить ему nullptr)? А значение dynamicArr? Почему?

staticArr: Нельзя, это константный указатель, жестко привязанный к адресу в стеке

dynamicArr: Можно, это обычная переменная указатель ей можно присвоить nullptr

## Что произойдёт, если забыть вызвать delete[] для динамического массива?

Произойдет утечка памяти

## Почему функция, принимающая int arr[], может работать и со статическим, и с динамическим массивом?

При передаче в функцию массив любого типа всегда «затухает» до обычного указателя int* поэтому функции всё равно, передали ей адрес начала стекового массива или адрес в куче
