# Задание 1

## Почему при передаче по значению исходные переменные не меняются?

При передаче по значению в функцию передается копия фактического значения аргумента, которая затем используется в конкретной области видимости самой функции. Все операции внутри функции выполняются над этой локальной копией. Исходная переменная в вызывающей функции остается нетронутой, поскольку находится в другой области памяти.

## В чем разница между указателем и ссылкой при передаче?

Оба способа меняют нашу переменную, но есть отличия:
- При использовании указателей необходимо явно использовать операторы адреса (&) при вызове функции и операторы разменования (*) внутри функции для доступа к значению. Указатель может быть nullptr и может быть переназначен на другой адрес в любой момент что может вызывать опередеделенные проблемы.
- Ссылки не могут быть нулевыми и не могут быть переназначены после инициализации.

## Когда безопаснее использовать ссылку, а когда указатель?
Согласно стандартну C++ нужно использовать ссылки когда это возможно, и указатели когда это необходимо. Ссылки безопаснее, указатели могут быть nullptr.

## Какой способ предпочтительнее для передачи больших объектов, если они не должны меняться?
Использовать константные ссылки, это будет наиболее безопасный и эффективный вариант.

## Происходит ли при передаче по значению/по указателю/по ссылке копия чего либо? Если да, то для каждого случая укажите чего копируется и как это влияет на результат.

|:----------------|:------------------------------------------------------------------------------|:---------------------------------------------------------------------------------------------------------|
