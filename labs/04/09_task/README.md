## 1. Что такое виртуальная таблица (vtable)?

Виртуальная таблица (vtable) — статическая таблица указателей на виртуальные функции, которую компилятор создаёт для каждого класса с виртуальными методами. Хранится в сегменте данных программы — одна таблица на класс. Каждый объект класса содержит скрытый указатель vptr, инициализируемый в конструкторе и указывающий на vtable своего класса. При вызове виртуального метода программа считывает vptr объекта, обращается к соответствующей записи в vtable и вызывает функцию по этому адресу.

## 2. Почему вызов area() является примером позднего связывания?

area() объявлена virtual в базовом классе. При вызове через указатель или ссылку на базовый класс компилятор не знает реальный тип объекта на этапе компиляции и не может подставить адрес функции статически. Вместо этого генерируется код, который во время выполнения считывает vptr объекта и вызывает функцию из vtable. Адрес функции определяется в момент выполнения это и есть позднее связывание.

## 3. Почему метод info() без virtual вызывается через раннее связывание?

Если info() не объявлена virtual, компилятор разрешает вызов статически, исходя только из статического типа выражения. При ptr->info(), где ptr имеет тип Figure*, компилятор подставляет адрес Figure::info() напрямую в машинный код. Никакого обращения к vtable во время выполнения не происходит. Реальный тип объекта, на который указывает ptr, при этом игнорируется.

## 4. Что произойдёт, если убрать override у переопределённого метода?

Если убрать override, программа будет работать точно так же — это не ошибка. Ключевое слово override является подсказкой компилятору: проверить, что данный метод действительно переопределяет виртуальный метод базового класса. Без override компилятор не проверяет это, и ошибкуможно не заметить метод просто скроет базовый, а не переопределит его, что приведёт к неожиданному поведению.

## 5. Почему абстрактный класс нельзя инстанцировать?

Абстрактный класс содержит хотя бы одну чисто виртуальную функцию (= 0). Для такой функции нет реализации в этом классе. Если бы компилятор разрешил создать объект абстрактного класса, то в vtable для этой функции стоял бы нулевой или недопустимый адрес, и любой вызов этой функции привёл бы к неопределённому поведению. Поэтому компилятор запрещает инстанцирование абстрактных классов.

## 6. Как запретить переопределение метода? (ключевое слово final)

Чтобы запретить переопределение метода в производных классах, используется спецификатор final

```cpp
virtual void info() const final;
```

Попытка переопределить такой метод в производном классе приведёт к ошибке компиляции. Ключевое слово final можно применять и ко всему классу запрещая наследование от него

```cpp
class Circle final : public Figure { ... };
```
