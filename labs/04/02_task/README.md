## Почему при работе с массивом указателей вызываются переопределённые методы?

При работе с std::vector<Figure*> каждый указатель хранит адрес реального объекта производного класса. Вызов виртуального метода через указатель разрешается через vtable: по адресу объекта считывается vptr, затем из vtable берётся адрес нужной функции

## Что происходит при работе с массивом объектов? Какая функция вызывается и почему?

При хранении объектов конкретного типа статический тип известен на этапе компиляции. Вызов методов разрешается статически вызывается метод того класса, чьим является объект. Полиморфизма нет, но и потери данных не происходит

Если бы Figure не был абстрактным и мы создали Figure arr[] = {Circle(5), Rectangle(3,4)}, объекты были бы срезаы

## Какое отношение это имеет к срезке (slicing)?

Срезка — потеря производной части объекта при копировании в переменную базового типа по значению. При массиве указателей срезки нет, т к копируется только указатель, а не объект. При массиве объектов базового класса происходит срезка: конструктор копирования Figure копирует только Figure-часть, данные производного класса теряются. Поэтому для полиморфизма нужны указатели или ссылки.
