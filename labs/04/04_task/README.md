## Почему размер изменился?

sizeof(Figure) больше sizeof(FigureNonVirtual), потому что Figure содержит виртуальные методы, а FigureNonVirtual нет. Наличие хотя бы одного virtual метода заставляет компиятор добавить в каждый объект скрытое поле — указатель на таблицу виртуальных функций. Это поле занимает sizeof(void*) байт (обычно 8 байт на 64-битной платформе).

## С чем связано увеличение размера класса при наличии виртуальных функций?

Компилятор добавляет vptr в каждый объект класса, у которого есть хотя бы один virtual метод. Этот указатель нужен для реализации полиморфизма во время выполнения. Размер самой vtable не входит в размер объекта — она хранится в сегменте данных программы и является общей для всех объектов одного класса. Размер объекта увеличивается ровно на один указатель.

## Что такое vptr и где он хранится?

vptr (virtual pointer) — скрытый указатель, который компилятор добавляет в каждый объект класса с виртуальными методами. Он хранится в начале объекта и указывает на vtable конкретного класса. При создании объекта конструктор инициализирует vptr. При создании производного объекта конструктор производного класса переустанавливает vptr на vtable производного класса.
