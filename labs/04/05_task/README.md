## Что такое срезка?

Срезка эьл явление, при котором производный объект копируется в переменную базового типа по значению. При этом копируется только часть объекта, соответствующая базовому классу. Помимо данных, при срезке теряется и полиморфизм: vptr в скопированном объекте указывает на vtable базового класса

## Какая часть объекта Circle5 сохранилась в объекте Figure5?

Сохранилась только часть, соответствующая Figure5: поле name и vptr, который теперь указывает на vtable Figure5. Поле radius, принадлежащее Circle5, было отброшено при копировании конструктор копирования Figure5 знает только о своих полях

## Почему при срезке теряется полиморфное поведение?

При срезке создаётся новый объект типа Figure5. Его vptr инициализируется конструктором Figure5 и указывает на vtable Figure5. Поэтому вызов любого виртуального метода на этом объекте разрешается через vtable Figure5
## Анализ log

В логе видно два разных результата area():
- Через Figure5* (указатель): возвращается M_PI * 25 ≈ 78.5398 — вызывается Circle5::area(), полиморфизм работает
- Через sliced (копия): возвращается 0.0 — вызывается Figure5::area(), срезка произошла
