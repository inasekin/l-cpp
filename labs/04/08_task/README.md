## Что такое vptr и где он хранится?

vptr (virtual pointer) — скрытый указатель, который компилятор добавляет в начало каждого объекта класса с виртуальными методами. Хранится в объекте по смещению 0. При создании объекта конструктор инициализирует vptr адресом vtable своего класса. При создании производного объекта конструктор производного класса перезаписывает vptr адресом vtable производного класса.

## Что такое vtable и что в ней содержится?

vtable (virtual function table) — статическая таблица указателей на функции, генерируемая компилятором для каждого класса с виртуальными методами. Хранится в сегменте данных программы. Содержит указатели на виртуальные методы класса в порядке их объявления: деструктор, затем остальные виртуальные функции. При переопределении метода в производном классе в его vtable заменяется соответствующий указатель.

## Почему у разных объектов одного класса vptr указывают на одну vtable?

vtable — это статический объект, общий для всего класса. Все экземпляры Circle должны вести себя одинаково при виртуальных вызовах, поэтому им нет смысла иметь разные таблицы. vptr в каждом объекте указывает на одну и ту же vtable класса. Это потверждается в программе: vptr у c1 и c2 совпадают, а vptr у Circle и Rectangle различаются.

## Анализ вывода программы

- c1 vptr == c2 vptr — оба Circle используют одну vtable
- c1 vptr != r vptr — Circle и Rectangle имеют разные vtable
- sizeof(Circle) = 40 байт (8 байт vptr + 24 байт std::string name + 8 байт double radius)
- sizeof(Rectangle) = 48 байт (8 байт vptr + 24 байт std::string name + 8 байт double width + 8 байт double height)
